## what i learned

token design is hard because incentives matter. easy to accidentally create:
- extraction (early buyers profit from later buyers)
- complexity (too many features, unclear purpose)
- disappointment (utility doesn't match expectations)

better to start minimal:
- clear single purpose (funding + signaling)
- no promises of returns
- upgrade path if needed
